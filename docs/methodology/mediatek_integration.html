2. MediaTek's AI Chip Design Methodology

Key Pillars

1. Heterogeneous AI-SoC Architecture

Dimensity integrates 4+ compute engines on one die: CPU (Arm Cortex), GPU (Arm Mali/Immortalis), APU (in-house AI), and DSP. AI workloads are dynamically scheduled across all engines by CorePilot — not a fixed "AI goes to NPU" mapping.

2. In-House APU Design

The APU (AI Processing Unit) is MediaTek's proprietary neural accelerator, designed in-house. Unlike competitors who license IP (e.g., Ceva, VeriSilicon), MediaTek designs its own NPU micro-architecture — optimized specifically for their SoC fabric and memory hierarchy.

3. AI-Enhanced Design Flow

MediaTek deploys AI across its chip design process: floorplanning (AI-assisted macro placement for heterogeneous dies), timing closure (ML prediction of critical paths), power optimization (RL-guided DVFS), and verification (AI-generated test vectors).

4. Edge-to-Cloud Continuum

Same AI architecture scales from Genio IoT (sub-watt edge AI) → Dimensity mobile (5W) → ASIC data center (100W+). Common software stack (NeuroPilot SDK) across all tiers.

MediaTek's differentiating insight: Don't build one AI accelerator. Build a family of accelerators that share the same architecture, software stack, and design methodology. The APU in a $5 IoT chip uses the same ISA as the APU in a $10,000 data center ASIC. This is what lunarch should learn — design once, scale everywhere.

4. CorePilot — MediaTek's Heterogeneous AI Scheduler

CorePilot is MediaTek's AI-driven heterogeneous multi-processing technology. It dynamically schedules workloads across CPU clusters (big/LITTLE), GPU, APU, and DSP based on:

InputDecision
Workload type (inference / training / DSP / graphics)Route to optimal compute engine
Thermal headroom (junction temperature)Throttle or boost specific engines
Battery level (mobile) / Power cap (IoT)Select low-power or high-performance engine
Memory bandwidth availabilityBatch or serialize concurrent AI tasks
Real-time latency SLAPrioritize tasks (camera > gaming > background AI)
What lunarch can adopt: A lunarch CorePilot that schedules agent AI tasks across v2 Orion cores + NPU + future accelerator IP. Each agent context maps to a compute engine, and the scheduler picks the optimal engine per agent task. This is the natural evolution of our 4-context hardware switching — add heterogeneous targets behind those contexts.
