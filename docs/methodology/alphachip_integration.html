4. AlphaChip-Enhanced lunarch GDS Flow

Integrating RL-based macro placement into lunarch's Python PD engine:

# Current lunarch GDS (fixed grid):
for i in range(n_cells):
    x = margin + (i % cols) * cell_width     # Fixed grid
    y = margin + (i // cols) * cell_height    # No optimization
    add_sref(x, y, cell_name)

# AlphaChip-enhanced lunarch GDS (RL placement):
from alpha_placer import RISCVMacroPlacer

placer = RISCVMacroPlacer(
    macros=['ICache', 'DCache', 'KV-cache', 'RegFile', 'STU'],
    netlist=parse_pycircuit_netlist('lunahan_v2_core.py'),
    pre_trained='tpu_checkpoint_20240815'
)
placements = placer.optimize(
    objectives=['wirelength', 'congestion', 'density'],
    iterations=200
)
for macro_name, (x, y, flip) in placements.items():
    add_sref(x, y, macro_name, flip=flip)  # RL-optimized location

6. How It Works in Practice

  1. Netlist → Graph: Parse lunarch pyCircuit RTL, extract SRAM macros as nodes, signal wires as edges. Build protocol buffer netlist.
  2. GNN Encoding: 3-layer Graph Neural Network encodes macro connectivity. Each node gets an 8-dim embedding.
  3. PPO Placement: RL agent sequentially places macros. State = current partial floorplan. Action = (x, y, orientation) for next macro.
  4. DREAMPlace: After macros placed, DREAMPlace analytically places standard cells respecting macro locations.
  5. Reward: Proxy wirelength × 1.0 + Density × 1.0 + Congestion × 0.5. Higher is better.
  6. Pre-training: Start from Google's TPU checkpoint (transfer from data-center to embedded RISC-V — works due to common macro placement patterns).
